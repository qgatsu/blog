# 導入
今回の記事のモチベーションを書く。なるべく固くなりすぎずに、以下の内容を盛り込む
・通常SHAPは特徴量寄与の表現としてかなりメジャーである
・ただ、カテゴリ列をone-hotすると列ごとのSHAPになるので、元のカテゴリそのものがどれくらい効いたかはつぶされる
・加えてこれを非エンジニアの第三者に説明する際に、「そもそもone-hot-encodingというのがあって・・・」から始めなければならず面倒
・これはgroupSHAPを行って、groupSHAP→通常SHAPとうまくつなぐことができれば説明しやすい

# そもそもSHAPとは
ここもなるべく優しく、ただ説明的になりすぎない。
・SHAPの基本的な考え方や思想、数理的な説明をおこなう
・数式を使った説明がいる場合は重要なものに絞って行う

# groupSHAPとは
さっきの説明と対比して、「通常SHAPと何が違うか」を説明。とくに
・group処理をどうしているか
・なぜ通常のSHAPにくらべて処理が重いのか

は数理的に説明したいです。
